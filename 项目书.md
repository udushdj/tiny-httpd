# 轻量级HTTP服务器与CGI引擎项目书

**目标：5天从零实现可演示的epoll+CGI Web服务器（ARM板部署 + LVGL状态面板）**

---

## 1. 项目概述

| 项目 | 说明 |
|------|------|
| **项目名称** | tiny-httpd（多线程HTTP/1.1服务器与CGI引擎） |
| **技术栈** | C语言、Linux系统调用（epoll、fork、exec、pipe、SO_REUSEPORT）、HTTP/1.1协议、LVGL v8、pthread、OpenSSL |
| **开发周期** | 5天，每天产出可运行的中间版本 |
| **开发环境** | WSL Ubuntu 22.04 (x86_64)，先本地编译验证 |
| **目标平台** | ARM 32-bit Linux 开发板（交叉编译部署） |
| **交叉编译器** | `arm-linux-gcc` 5.4.0 (`/home/wz/usr/local/arm/5.4.0/usr/bin/arm-linux-gcc`) |
| **部署方式** | 交叉编译 → SCP 上传到开发板 → 开发板运行 |

### 运行拓扑

```
┌──────────────────────┐          HTTP 请求          ┌──────────────────────┐
│   你的电脑 (客户端)    │ ──────────────────────────→  │  ARM 开发板 (服务器)   │
│   Windows 浏览器/curl │ ←────────────────────────── │  IP: 192.168.x.x      │
│   192.168.x.100       │        HTTP 响应             │  端口: 8080            │
└──────────────────────┘                              │                        │
                                                      │  ┌─────────────────┐  │
                                                      │  │  LVGL 状态面板   │  │
                                                      │  │  (板载屏幕)     │  │
                                                      │  └─────────────────┘  │
                                                      └──────────────────────┘
```

### 核心功能

- 基于 epoll + SO_REUSEPORT 的多线程线程池架构（worker线程数 = CPU核心数）
- HTTP/1.1 请求解析（GET / POST，支持 Content-Length）
- 静态文件服务（MIME 类型映射、路径穿越防护）
- CGI 动态请求（fork + exec、管道通信、环境变量传递）
- 连接超时管理与资源回收（僵尸进程回收）
- **`/api/status` 接口**：返回 JSON 格式服务器统计信息
- **LVGL 服务端状态面板**：在开发板屏幕上实时显示服务器运行状态
- **日志文件系统**：所有请求信息 + 服务器运行事件持久化写入日志文件，支持日志轮转
- **编译宏开关**：通过宏控制调试输出、测试模式、日志级别等，生产/开发灵活切换
- **多线程压力测试工具**：内建多线程 HTTP 客户端，可配置并发数/请求数，验证服务器抗压能力

## 2. 模块架构与数据流

### 多线程线程池架构

```
                              Linux 内核 (SO_REUSEPORT)
                         ┌───────┬───────┬───────┬───────┐
                         │       │       │       │       │
                    listen_fd listen_fd listen_fd listen_fd
                         │       │       │       │       │
                    ┌────▼──┐ ┌─▼──────┐┌─▼──────┐┌─▼──────┐
                    │Worker0│ │Worker1 ││Worker2 ││Worker3 │
                    │epoll  │ │epoll   ││epoll   ││epoll   │
                    │accept │ │accept  ││accept  ││accept  │
                    │read   │ │read    ││read    ││read    │
                    │parse  │ │parse   ││parse   ││parse   │
                    │route  │ │route   ││route   ││route   │
                    │write  │ │write   ││write   ││write   │
                    └───┬───┘ └───┬────┘└───┬────┘└───┬────┘
                        │         │         │         │
                        └─────────┼─────────┼─────────┘
                                  │         │
                          ┌───────▼──┐ ┌───▼──────────┐
                          │ 共享状态  │ │   logger 模块  │
                          │ stats    │ │ (pthread_mutex)│
                          │ + mutex  │ │ access/server  │
                          └───────┬──┘ │ /error.log     │
                                  │    └───────────────┘
                          ┌───────▼──────────┐
                          │  LVGL UI 主线程    │
                          │  lv_timer(200ms)  │
                          │  仪表盘 / 日志     │
                          └──────────────────┘
```

**关键设计**：
- 所有 worker 线程共享同一个端口，通过 `SO_REUSEPORT` 让内核做负载均衡
- 每个 worker 线程完全独立：自己的 epoll 实例、自己的连接数组，**无锁竞争**
- worker 线程只通过 `pthread_mutex` 更新全局 `server_stats_t` 和 `logger_t`
- LVGL UI 线程只读 shared_stats，定时刷新屏幕
- 线程数默认为 `sysconf(_SC_NPROCESSORS_ONLN)`，也可通过 `-t` 参数指定

### 单 Worker 线程内部数据流（核心：server.c 无锁事件循环）

```
客户端请求 → 监听 socket → epoll 事件循环（每个 worker 线程独立）
  ├─ 新连接           → accept() → 加入本线程 epoll (EPOLLIN, 非阻塞)
  │                     → stats.conn_count++（加锁）
  ├─ 客户端可读       → 读取数据 → HTTP 解析状态机
  │    └─ 完整请求 → 路由判断
  │         ├─ 静态文件  → 安全检查 → stat → mmap → EPOLLOUT
  │         ├─ CGI 请求  → pipe → fork → 子进程 exec → 管道读取 → EPOLLOUT
  │         └─ /api/status → 读取 stats → 构造 JSON 响应 → EPOLLOUT
  ├─ 客户端可写       → 非阻塞 write → 完成后关闭 → stats.conn_count--（加锁）
  │                     → log_access() 写入请求日志
  └─ 超时检查         → 清理空闲连接 + kill CGI + 回收僵尸
```

### LVGL 界面数据流 (gui.c)

```
lv_timer(200ms) 定时器
  │
  ├─ pthread_mutex_lock(&stats.lock)
  ├─ 读取 stats.conn_count, stats.req_total, stats.uptime
  ├─ 读取系统信息 (/proc/stat, /proc/meminfo)
  ├─ pthread_mutex_unlock(&stats.lock)
  │
  └─ 更新 LVGL 控件
       ├─ 仪表盘区域: 连接数、请求数、运行时间
       ├─ 系统信息区域: CPU使用率、内存使用量
       ├─ 线程状态区域: 各 worker 线程的连接数/请求数
       └─ 请求日志区域: 最近 20 条请求记录
```

## 3. 核心数据结构

```c
#define MAX_CONN   1024
#define BUF_SIZE   8192
#define TIMEOUT    30
#define MAX_EVENTS 1024
#define DOCROOT    "./www"
#define LOG_MAX    128         /* 内存环形日志最大条数 */

/* 连接状态 */
enum { STATE_READING, STATE_WRITING, STATE_CGI };

/* 解析器状态 */
enum { PARSE_REQ_LINE, PARSE_HEADERS, PARSE_BODY, PARSE_DONE };

/* HTTP 方法 */
enum { HTTP_GET, HTTP_POST, HTTP_UNKNOWN };

/* 日志级别 */
typedef enum {
    LOG_DEBUG,       /* 调试信息，仅 DEBUG 模式输出 */
    LOG_INFO,        /* 常规信息（启动、连接、请求摘要） */
    LOG_WARN,        /* 警告（超时、CGI异常退出） */
    LOG_ERROR        /* 错误（socket失败、fork失败） */
} log_level_t;

/* 日志目标 */
typedef enum {
    LOG_TARGET_CONSOLE,  /* 仅控制台（stdout/stderr） */
    LOG_TARGET_FILE,     /* 仅文件 */
    LOG_TARGET_BOTH      /* 控制台 + 文件（默认） */
} log_target_t;

/* 连接上下文 */
typedef struct {
    int         fd;
    int         state;
    char        rbuf[BUF_SIZE];
    size_t      rpos, rlen;
    char        wbuf[BUF_SIZE];
    size_t      wpos, wlen;
    time_t      last_active;

    struct http_parser  parser;
    struct http_request request;

    int         cgi_fd;
    pid_t       cgi_pid;
    char        cgi_output[BUF_SIZE];
    size_t      cgi_olen;
} conn_t;

/* HTTP 解析器 */
typedef struct http_parser {
    int         state;
    int         content_len;
    int         body_read;
    char        line_buf[2048];
    int         line_pos;
} http_parser;

/* HTTP 请求 */
typedef struct http_request {
    int         method;
    char        uri[1024];
    char        query[1024];
    char        version[16];
    char        headers[32][2][256];
    int         header_cnt;
    char       *body;
    size_t      body_len;
} http_request;

/* 请求日志条目 */
typedef struct {
    char   method[8];
    char   uri[256];
    int    status;
    char   client_ip[32];
    time_t timestamp;
} log_entry_t;

/* 线程池配置 */
#define MAX_WORKERS 64             /* 最大 worker 线程数 */

/* 每个 worker 线程的参数 */
typedef struct {
    int             id;            /* worker 编号 0..N-1 */
    int             port;          /* 监听端口 */
    server_stats_t *stats;         /* 全局统计 */
    logger_t       *logger;        /* 全局日志 */
    int             running;       /* 运行标志 */
} worker_args_t;

/* 服务器共享状态（所有 worker 线程写入，LVGL 线程读取） */
typedef struct {
    pthread_mutex_t lock;
    int     conn_count;            /* 当前活跃连接总数 */
    int     req_total;             /* 累计请求总数 */
    time_t  start_time;            /* 服务器启动时间 */
    log_entry_t log[LOG_MAX];      /* 环形日志缓冲区 */
    int     log_head;              /* 日志写入位置 */
    int     log_count;             /* 日志总条数 */
    int     worker_conns[MAX_WORKERS];  /* 每个 worker 的重连接数 */
    int     worker_reqs[MAX_WORKERS];   /* 每个 worker 的请求数 */
    int     worker_count;          /* 实际 worker 线程数 */
} server_stats_t;

/* 日志系统上下文 */
typedef struct {
    FILE       *access_fp;       /* access.log 文件指针 */
    FILE       *server_fp;       /* server.log 文件指针 */
    FILE       *error_fp;        /* error.log 文件指针 */
    char        log_dir[256];    /* 日志目录路径 */
    log_level_t min_level;       /* 最低记录级别 */
    log_target_t target;         /* 输出目标 */
    size_t      max_size;        /* 单文件最大字节（超过则轮转） */
    pthread_mutex_t lock;        /* 线程安全 */
} logger_t;
```

## 3.5 日志文件系统设计

### 设计目标

将服务器的所有关键事件持久化到磁盘，方便事后排查问题、统计分析和审计。

### 日志文件分三类

| 文件 | 路径 | 内容 | 示例 |
|------|------|------|------|
| **access.log** | `logs/access.log` | 每个HTTP请求的摘要行 | `[2026-06-02 14:32:10] 192.168.1.100 GET /index.html 200 12ms` |
| **server.log** | `logs/server.log` | 服务器生命周期事件 | `[INFO] 服务器启动, 端口 8080` `[INFO] CGI进程 fork 成功 pid=1234` |
| **error.log** | `logs/error.log` | 严重错误（单独文件方便告警） | `[ERROR] accept() 失败: Too many open files` |

### 日志格式

```
access.log:
[YYYY-MM-DD HH:MM:SS] 客户端IP 方法 URI 状态码 处理耗时ms [可选:User-Agent]

server.log:
[YYYY-MM-DD HH:MM:SS] [LEVEL] 描述信息

error.log:
[YYYY-MM-DD HH:MM:SS] [ERROR] 函数名() 失败: 错误原因 (errno描述)
```

### 日志轮转策略

- 单文件超过 `LOG_MAX_SIZE`（默认 10MB）时自动轮转
- 轮转规则：`access.log` → `access.log.1` → `access.log.2`（保留最近3份）
- 轮转时机：每次 `log_write()` 前检查文件大小，超阈值则触发
- 轮转过程：关闭当前文件 → 重命名旧文件 → 打开新文件（日志不中断）

### 线程安全

- 日志写入使用独立的 `pthread_mutex_t`，不影响 `server_stats_t` 的锁
- 写入策略：立即 `fflush()`，保证崩溃时不丢失关键日志

### logger 模块 API

```c
/* 初始化日志系统 */
logger_t* logger_init(const char *log_dir, log_level_t level, log_target_t target);

/* 写入一条日志（线程安全） */
void log_write(logger_t *l, log_level_t level, const char *fmt, ...);

/* 写入请求日志（access.log专用） */
void log_access(logger_t *l, const char *ip, const char *method,
                const char *uri, int status, int elapsed_ms);

/* 写入服务器事件日志 */
void log_server(logger_t *l, log_level_t level, const char *fmt, ...);

/* 写入错误日志（同时写入 server.log 和 error.log） */
void log_error(logger_t *l, const char *func_name, const char *fmt, ...);

/* 检查并执行日志轮转 */
void log_rotate_check(logger_t *l);

/* 关闭日志系统（flush + close） */
void logger_close(logger_t *l);
```

## 3.6 编译宏开关设计

### 设计目标

通过编译期宏实现"一套代码，多种编译模式"，无需修改源码即可切换行为。

### 宏定义一览

```c
/* ──── 编译模式控制 ──── */
#define DEBUG         /* 调试模式：开启详细日志输出、assert、EAGAIN计数 */
#define TEST_MODE     /* 测试模式：固定端口 + 固定DOCROOT + 自动退出 */

/* ──── 功能开关 ──── */
#define ENABLE_LOG        1  /* 是否启用日志文件（0=仅控制台） */
#define ENABLE_CGI        1  /* 是否启用CGI支持（0=纯静态） */
#define ENABLE_LVGL       1  /* 是否启用LVGL面板（0=无头模式） */

/* ──── 参数配置 ──── */
#define LOG_DIR         "logs"       /* 日志目录 */
#define LOG_MAX_SIZE    (10*1024*1024) /* 日志轮转阈值 10MB */
#define TEST_PORT       18080         /* 测试模式固定端口 */
#define TEST_DOCROOT     "./www"      /* 测试模式固定DOCROOT */
```

### 模式对照表

| 场景 | 编译命令 | 效果 |
|------|---------|------|
| **本地开发调试** | `make debug` | `-DDEBUG`：控制台打印所有日志（LOG_DEBUG级别） |
| **本地测试** | `make test` | `-DTEST_MODE`：固定端口18080，静态文件直出，100请求后自动退出 |
| **生产部署（ARM+LVGL）** | `make arm` | 无 `DEBUG/TEST_MODE`，日志写文件，LVGL面板启用，LOG_INFO级别 |
| **无头模式（ARM无屏幕）** | `make headless` | `-DENABLE_LVGL=0`：ARM部署但无GUI，日志写文件 |
| **纯静态服务器** | `make static` | `-DENABLE_CGI=0`：禁用CGI，适合资源受限场景 |

### 代码中使用方式

```c
/* DEBUG模式：打印详细的读/写字节数 */
#ifdef DEBUG
    printf("[DEBUG] fd=%d read %d bytes, EAGAIN=%d\n", c->fd, n, eagain_count);
#endif

/* TEST_MODE：请求计数器，达到阈值自动退出 */
#ifdef TEST_MODE
    static int test_req_count = 0;
    if (++test_req_count >= 100) {
        printf("[TEST] 已完成100个测试请求，自动退出\n");
        exit(0);
    }
#endif

/* 功能开关：编译期裁剪代码 */
#if ENABLE_LOG
    log_access(g_logger, ip, method, uri, status, elapsed);
#else
    printf("[%s] %s %s %d %dms\n", ip, method, uri, status, elapsed);
#endif

#if ENABLE_CGI
    if (strncmp(uri, "/cgi-bin/", 9) == 0) {
        exec_cgi(epfd, c);
    } else
#endif
    {
        serve_file(epfd, c);
    }
```

### Makefile 对应目标

```makefile
# 标准编译（x86_64，LOG_INFO级别，日志写文件）
$(TARGET): $(OBJS)
	$(CC) $(CFLAGS) -DENABLE_LOG=1 -DENABLE_CGI=1 -o $@ $^ $(LDFLAGS)

# 调试编译（控制台DEBUG输出）
debug: CFLAGS += -DDEBUG -DENABLE_LOG=0 -O0 -g
debug: $(TARGET)

# 测试编译（TEST_MODE，100请求自动退出）
test: CFLAGS += -DTEST_MODE -DDEBUG -O0 -g
test: $(TARGET)

# ARM生产部署
arm:
	$(CC_ARM) $(ARM_CFLAGS) -DENABLE_LOG=1 -DENABLE_CGI=1 -DENABLE_LVGL=1 \
		-o $(TARGET) $(ARM_SRCS) $(ARM_LIBS)

# 无头模式（ARM + 日志 + 无LVGL）
headless:
	$(CC_ARM) $(ARM_CFLAGS) -DENABLE_LOG=1 -DENABLE_CGI=1 -DENABLE_LVGL=0 \
		-o $(TARGET) $(ARM_SRCS) -lpthread -lm

# 纯静态服务器（无CGI）
static:
	$(CC) $(CFLAGS) -DENABLE_LOG=1 -DENABLE_CGI=0 -o $(TARGET) $(SRCS) $(LDFLAGS)
```

## 4. 五日开发计划

### Day 1：epoll 事件循环 + 非阻塞回显服务器 + 多线程框架

**目标**：本地编译通过，WSL 中用 nc 验证回显；扩展到多 worker 线程。

| # | 任务 | 说明 |
|---|------|------|
| 1.1 | 创建监听 socket | `socket → setsockopt(SO_REUSEPORT) → bind → listen`，绑定 8080，设非阻塞 |
| 1.2 | epoll 事件循环 | `epoll_create1 → epoll_ctl(ADD listen) → epoll_wait` |
| 1.3 | accept 新连接 | 循环 accept 至 EAGAIN，设非阻塞，注册 EPOLLIN |
| 1.4 | 读取与回显 | EPOLLIN → read → 拷贝到 wbuf → EPOLLOUT |
| 1.5 | 非阻塞写入 | EPOLLOUT → write → EAGAIN 继续 / 发完关闭 |
| 1.6 | 资源回收 | EPOLLERR/EPOLLHUP 时关闭连接 |
| 1.7 | **SO_REUSEPORT** | `setsockopt(listen_fd, SOL_SOCKET, SO_REUSEPORT, &opt, sizeof(opt))` |
| 1.8 | **多线程启动** | `main()` 中 `pthread_create` 创建 N 个 worker 线程，每个独立 `server_worker()` |

**产出**：`src/server.c` (~300行)

**验证**：
```bash
# WSL 终端1
make && ./tiny-httpd -p 8080 -t 4
# [INFO] 启动 4 个 worker 线程, 端口 8080

# WSL 终端2 (多开几个同时连)
nc 127.0.0.1 8080
hello       # 输入
hello       # 回显
```

**完成标志**：
- [x] nc 回显正常
- [x] 多客户端同时连接，分布在不同 worker 线程
- [x] 客户端断开不崩溃
- [x] `-t 4` 参数控制 worker 线程数

### 5.6 多线程入口 (main.c)

```c
#include "server.h"
#include "gui.h"

server_stats_t g_stats;  /* 全局共享状态 */
logger_t      *g_logger;  /* 全局日志 */

int main(int argc, char *argv[])
{
    int port       = 8080;
    int workers    = sysconf(_SC_NPROCESSORS_ONLN); /* 默认 = CPU 核心数 */
    char *docroot  = "./www";
    char *logdir   = "logs";

    /* 解析命令行: -p port -t threads -d docroot -l logdir */
    parse_args(argc, argv, &port, &workers, &docroot, &logdir);

    /* 初始化日志系统 */
    g_logger = logger_init(logdir, LOG_INFO, LOG_TARGET_BOTH);

    /* 初始化共享状态 */
    pthread_mutex_init(&g_stats.lock, NULL);
    g_stats.start_time   = time(NULL);
    g_stats.worker_count = workers;

    /* 创建 N 个 worker 线程 */
    pthread_t *tids = malloc(sizeof(pthread_t) * workers);
    worker_args_t *wargs = malloc(sizeof(worker_args_t) * workers);

    for (int i = 0; i < workers; i++) {
        wargs[i] = (worker_args_t){
            .id = i, .port = port,
            .stats = &g_stats, .logger = g_logger, .running = 1
        };
        pthread_create(&tids[i], NULL, server_worker, &wargs[i]);
    }

    log_server(g_logger, LOG_INFO, "启动 %d 个 worker 线程, 端口 %d", workers, port);

#if ENABLE_LVGL
    /* 主线程进入 LVGL 事件循环（阻塞） */
    gui_init(&g_stats);
#else
    /* 无头模式：等待所有 worker 线程 */
    for (int i = 0; i < workers; i++) pthread_join(tids[i], NULL);
#endif

    logger_close(g_logger);
    free(tids); free(wargs);
    return 0;
}
```

### 5.8 压力测试工具 (test/stress.c) —— 多线程 HTTP 客户端

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <pthread.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <unistd.h>
#include <time.h>
#include <sys/time.h>

#define MAX_REQ 100000

typedef struct {
    int    id;          /* 线程编号 */
    char   host[64];    /* 目标 IP */
    int    port;        /* 目标端口 */
    int    total;       /* 本线程要发送的请求数 */
    int    success;     /* 成功计数 */
    int    fail;        /* 失败计数 */
    double total_ms;    /* 总耗时(ms) */
    double min_ms;      /* 最小耗时 */
    double max_ms;      /* 最大耗时 */
} stress_thread_t;

static double now_ms(void)
{
    struct timeval tv;
    gettimeofday(&tv, NULL);
    return tv.tv_sec * 1000.0 + tv.tv_usec / 1000.0;
}

void *stress_worker(void *arg)
{
    stress_thread_t *st = (stress_thread_t *)arg;
    char request[512];
    snprintf(request, sizeof(request),
        "GET / HTTP/1.1\r\nHost: %s\r\nConnection: close\r\n\r\n",
        st->host);

    st->min_ms = 1e9;
    st->max_ms = 0;

    for (int i = 0; i < st->total; i++) {
        int fd = socket(AF_INET, SOCK_STREAM, 0);
        struct sockaddr_in addr = {
            .sin_family = AF_INET,
            .sin_port   = htons(st->port)
        };
        inet_pton(AF_INET, st->host, &addr.sin_addr);

        double start = now_ms();

        if (connect(fd, (struct sockaddr*)&addr, sizeof(addr)) == -1) {
            st->fail++;
            close(fd);
            continue;
        }

        send(fd, request, strlen(request), 0);

        char buf[4096];
        int total_read = 0;
        while (1) {
            int n = recv(fd, buf, sizeof(buf), 0);
            if (n <= 0) break;
            total_read += n;
        }
        close(fd);

        double elapsed = now_ms() - start;
        st->total_ms += elapsed;
        if (elapsed < st->min_ms) st->min_ms = elapsed;
        if (elapsed > st->max_ms) st->max_ms = elapsed;

        if (total_read > 0) st->success++;
        else st->fail++;
    }
    return NULL;
}

int main(int argc, char *argv[])
{
    char   *host   = "127.0.0.1";
    int     port   = 8080;
    int     total  = 1000;
    int     conc   = 10;    /* 并发线程数 */

    /* 解析 -h -p -n -c 参数 */
    for (int i = 1; i < argc; i++) {
        if (!strcmp(argv[i], "-h") && i+1<argc) host = argv[++i];
        else if (!strcmp(argv[i], "-p") && i+1<argc) port = atoi(argv[++i]);
        else if (!strcmp(argv[i], "-n") && i+1<argc) total = atoi(argv[++i]);
        else if (!strcmp(argv[i], "-c") && i+1<argc) conc = atoi(argv[++i]);
    }

    printf("[STRESS] 目标: http://%s:%d/\n", host, port);
    printf("[STRESS] 总请求: %d, 并发: %d 线程\n", total, conc);

    stress_thread_t *sts = calloc(conc, sizeof(stress_thread_t));
    pthread_t *tids      = malloc(sizeof(pthread_t) * conc);

    double t_start = now_ms();

    /* 每个线程分配 total/conc 个请求 */
    for (int i = 0; i < conc; i++) {
        sts[i].id = i;
        strcpy(sts[i].host, host);
        sts[i].port  = port;
        sts[i].total = total / conc + (i < total % conc ? 1 : 0);
        pthread_create(&tids[i], NULL, stress_worker, &sts[i]);
    }

    /* 汇总结果 */
    int ok = 0, fail = 0;
    double total_ms = 0, min_ms = 1e9, max_ms = 0;
    for (int i = 0; i < conc; i++) {
        pthread_join(tids[i], NULL);
        ok       += sts[i].success;
        fail     += sts[i].fail;
        total_ms += sts[i].total_ms;
        if (sts[i].min_ms < min_ms) min_ms = sts[i].min_ms;
        if (sts[i].max_ms > max_ms) max_ms = sts[i].max_ms;
    }

    double elapsed = now_ms() - t_start;
    double qps = (ok + fail) / (elapsed / 1000.0);

    printf("[STRESS] 完成: %d/%d\n", ok + fail, total);
    printf("[STRESS] 耗时: %.2fs, QPS: %.1f\n", elapsed/1000.0, qps);
    printf("[STRESS] 成功: %d, 失败: %d\n", ok, fail);
    if (ok > 0)
        printf("[STRESS] 最小: %.0fms, 平均: %.0fms, 最大: %.0fms\n",
               min_ms, total_ms / (ok+fail), max_ms);

    free(sts); free(tids);
    return fail > 0 ? 1 : 0;
}
```

## 开发流程总结

```
Day 1: epoll + SO_REUSEPORT + 多线程  → nc 验证回显, 4 worker并发
Day 2: HTTP解析 + 静态文件              → 浏览器可访问网页, 多worker无锁
Day 3: CGI引擎 + 交叉编译               → 开发板运行, PC远程访问, CGI正常
Day 4: 健壮性 + 日志 + 压力测试工具      → 完整HTTP服务器 + 日志文件持久化
Day 5: LVGL面板 + 部署脚本              → 屏幕实时多worker面板, 一键部署
```

**核心原则**：
1. 本地 WSL 编译验证通 → 再交叉编译部署
2. 每个 worker 线程完全独立（各自 epoll + 连接数组），只在更新全局状态时加锁
3. SO_REUSEPORT 让内核做负载均衡，无需应用层任务分发
4. HTTP 引擎与 LVGL 界面分离开发，共享 `server_stats_t` 通信
5. 善用 `stress_test` 压力测试工具验证多线程并发正确性
6. 善用 `strace` / `valgrind` / `curl -v` 调试
